Getting started

About Thymio

Thymio is a mobile robot designed for education. It has various sensors and actuators that can be programmed through different software.

For more information, visit www.thymio.org

About VPL

VPL is a block-based visual programming language. You can program behaviours for Thymio by associating conditions/events to actions.

Connecting a Thymio

Aseba 1.6.1 (and previous)

Plug your Thymio to your computer (with a USB cable or with a dongle), then turn it on.
Launch the Thymio Web Bridge inside your Aseba folder.

Thymio should now be detected by VPL 3, you are ready to go!

Thymio Suite

Plug your Thymio to your computer (with a USB cable or with a dongle), then turn it on.
Launch the Thymio Suite, Thymio should now be detected by VPL 3, you are ready to go!

An icon inside the console shows the connection status as follows:

A Thymio is connected

Creating a program

To create a program simply drag one or more events from the left list of events to the left side of a program line. Then associate one or more actions you want to happen in the right side of the line, taking them from the right choice on the screen. A line must always contain at least one event and one action. Two lines cannot have the same triggering event and conditions with the same settings. If two lines produce conflicting actions, the result is undefined.

Executing a program

The green PLAY button means the code has no errors, a robot is connected and the code in the interface is not yet inside the robot.

When you hit PLAY the code is sent to the robot, the button changes and you can REPLAY the same code whenever you want.

As soon as the code has been modified (or you clicked on STOP) the button goes back to the PLAY state.

Conditions and action blocks

Events are colored in light orange and actions in light blue. To better distinguish them also in blank/white printing, event blocks have in addition the right-top corner colored in orange to distinguish them from actions:

Two types of conditions

On the left side of a program line you can associate, to a given event, as many condition as you want. Conditions define if the event is valid or not. In the case of the line below one can say “when you push the right button (event), if the bottom color of Thymio is magenta (condition), then move forward-right (action)”. Conditions  are displayed in a slightly smaller size to indicate the difference with the main event.


While “push the button” is a real event, some others icons (LED stated and internal variables) are representing internal states and not real events. In case one of these icons is set first, the program line will display all conditions at the same level, and all conditions will be considered at the same level. In the case of the line below one can say “when we come to the situation that the bottom color if Thymio is magenta (condition) and the right button is pressed (condition) at the same time then move forward-right (action)”.

Note from the given example that switching events and conditions results in very different behaviors.

All LEDS and internal states icons can only be conditions and never events. All sensor readings and the clock mechanism can be used as events.

Events and states

Buttons

Sensor(s) used: five capacitive touch buttons on top of Thymio.

How to use it: this event detects if a button is pressed.

Orange:  button is pressed.

Grey: button is not considered.

Use this icon as an event to trigger an action when you push a button.

Remote control

Sensor(s) used: infrared remote control receiver.

How to use it: this event detects if a button is pressed on the remote control.

Orange:  button is pressed.

Grey: button is not considered.

Use this icon as an event to trigger an action when you push a button on a remote control.

Start of the program

This event does not use a specific sensor of the Thymio.

How to use it: this event is triggered at the start of the program, as you hit the PLAY button within VPL. You can use it to set an action Thymio will perform at the start of the program. For instance if from the beginning you want your robot to be red.

On the side

Sensor used: 3 axis accelerometer.

How to use it: This event detects Thymio’s orientation. You can detect if Thymio is standing on its side. Both sides are detected. You can use it do trigger an action when the robot is placed by the user in this position.

On the back

Sensor used: 3 axis accelerometer.

How to use it: This event detects Thymio’s orientation. You can detect if Thymio is completely upside down. You can use it to trigger an action when the robot is placed by the user in this position.

Accelerometer

Sensor(s) used: 3 axis accelerometer.

How to use it: This event detects Thymio’s orientation, measuring any position on the three different axis.

Move the circular teal handle around the circle to choose the desired rotation. The orange arrow indicates the direction of gravity.

Tap

Sensor(s) used: 3 axis accelerometer.

How to use it: this event allows you to know when Thymio senses a shock in any direction. A tap with your hands is detected as a shock.

Clap

Sensor(s) used: microphone.

How to use it: this event allows you to know when a loud sound is detected by Thymio. A tap of the robot can also trigger this event. A noisy environment can trigger this event in an uncontrolled way.

Ground detection

Sensor(s) used: ground proximity sensors.

How to use it: This event detects if there is a surface under Thymio.

Red /White : a ground is detected

Black: no ground (or a black ground) is detected

Note that a transparent or a black (matte) surface won’t be detected.

Ground sensors

Sensor(s) used: ground proximity sensors.

How to use it:  This event allows you to separately read what the left and right ground sensors are detecting. They detect a black or a white surface under Thymio. You can easily create a line following behaviour.

White/red: the sensor detects a white surface

Black: the sensor detects a black surface

Grey: the sensor is being ignored.

Note that black can also mean the absence of surface.

Tunable ground sensors

Sensor(s) used: ground proximity sensors.

How to use it: Ground sensors are proximity infrared sensors that detect the reflection of the surface they hit.

Red/White: the sensor detects a reflection of a clear surface.

Black: the sensor does not detect any reflection, like on matte black surfaces, or only a weak reflection.

Grey: the sensor is being ignored.

You can tune the distinction between detection and no detection or between white and black surfaces using a threshold (slider) to know which range of blacks or whites is detected under the Thymio. The range of detected blacks or whites is illustrated on the surface under the sensors.

The threshold value goes from 0 to 1000.

Note that a very reflective surface will be considered white and the absence of surface is considered as black.

Proximity sensors

Sensor(s) used: five proximity sensors at the front and two on the back.

How to use it: proximity infrared sensors measure the reflection of the surface in front of them. If there is reflection, this means that there is a close or very reflective obstacle in front of them. No reflection means either a black obstacle or no obstacle at all.

White/red: an obstacle is detected (reflecting on the sensor)

Black:  no obstacle is detected (nothing is reflecting)

Grey: the sensor is being ignored.

Note that a transparent or a black (matte) object will be considered as absent.

Tunable proximity sensors

Sensor(s) used: five proximity sensors at the front or two on the back.

How to use them: proximity infrared sensors measure the reflection of the surface they hit. This way we know if there are obstacles around Thymio.

White/red: an obstacle is detected (reflecting on the sensor)

Black:  no obstacle is detected (nothing is reflecting)

Grey: the sensor is being ignored.

You can tune the distinction between obstacle detection and no detection using a threshold (slider). This will indirectly specify if the obstacle will be detected close or far. The higher the threshold the closer to the sensor the detection is being done.

The threshold value goes from 1400 to 3300.

Note that a transparent or a black (matte) object will be considered as absent.
Different gray shades will be detected or not depending on the threshold.

Timer up

This event does not use a specific sensor of the Thymio. It reacts to a timer inside Thymio.

How to use it: this event detects when the timer (set in the actions) is over.

Compare

This event does not use a specific sensor of the Thymio. It reads an internal state, a 8-bit variable inside Thymio. It can be used only as a condition, not as a main event.

How to use it: this event is reading the value of an internal variable, represented on the led circle on Thymio. On the left you choose among a list of comparators:

= Equal to
≥ Bigger than

> Strictly bigger than

≤ Smaller than

< Strictly smaller than

On the right you choose a number from 0 to 255. This number is displayed on the circular LEDs as a binary number based on the binary code illustrated in the image on the left.



LED check

This event does not use a specific sensor of the Thymio.  It reads the status of the 8 LEDs around Thymio’s buttons. It can be used only as a condition, not as a main event.

How to use it: this block allows you to know if these LEDs are on or off.

Yellow:  LED is on.

Dark grey: LED is off.

You can use this to create patterns or even basic state machines by associating the position (on or off) of the LEDs to a specific behaviour. Since each LED can be either on or off there are 256 possible combinations.

Selective LEDs check

This event does not use a specific sensor of the Thymio. It reads the status of the 8 LEDs around Thymio’s buttons.  It can be used only as a condition, not as a main event.

How to use it: this block allows you to know if some of the LEDs are on or off. You can choose to read only some LEDs, ignoring others.

Yellow:  LED is on.

Dark grey: LED is off.

Light Grey : LED is ignored

This allows to detect partial configurations of LEDs, distinguishing various patterns at the same time or making subgroups of LEDs.

State: colors

This event does not use a specific sensor of the Thymio.  It reads the status of the RGB LEDs displayed on Thymio, two on top and two on the bottom. It can be used only as a condition, not as a main event.

How to use it: use this block to check if Thymio RGB LEDs are set to a specific color. You can separately read the top and the bottom RGB LEDs.

This block allows the creation of state machines as you can read the color (set by an action block) and then use it as a condition for an event. For instance you, when you press a button you can change the color to green if the robot is red (checked with this icon) and change to red if the robot is green.

Actions

Directions

Actuator: setting the speed of the wheels

How to use it:  this action allows you to set a direction of movement for Thymio. Choose any arrow to make Thymio move or choose the center button to make it stop.

Note: with this block Thymio will always have the same speed (around 6-7 cm/s).

Wheels speed

Actuator: wheels motors

How to use it: this action allows you to set the speed of each wheel to make Thymio move. Each slider represents a wheel. The position on the slider indicates the speed of the wheel which goes from 0 to 500 (forward) and from 0 to -500 (backward). A speed of 500 is around 17cm/s.

To turn to a side simply set the speed of the opposite side wheel to a higher speed than the other. Depending on both speeds Thymio will make narrower or wider curves.

Dark grey point: Speed is equivalent on both wheels.

Red square: Thymio is stopped.

Set Colors (basic)

Actuator: top and bottom RGB LED

How to use it: use this action to set a specific color for the Thymio RGB LEDs.

You can separately set the top and the bottom RGB LEDs.

Set Colors (advanced)

Actuator: top and bottom RGB LED

How to use them: use this action to set a color for Thymio’s RGB LEDs. You can separately set the top and the bottom RGB LEDs.

In advanced mode you can precisely set the shade you want by combining the RED, GREEN and BLUE led values following the additive color system. When equal parts of red, blue, and green are added to one another, white light is created:

Music

Actuator: loud speaker

How to use it: use this action to make a melody with Thymio. Each blue line corresponds to a tone and you can choose the duration of the note by clicking on it:


Black: 1 unit

White: 2 units

Empty: 4 units silence

Music With Tempo

Actuator: loud speaker

How to use it: use this action to make a melody with Thymio. Each blue line corresponds to a tone and you can choose the duration of the note by clicking on it:


Black: 1 unit

White: 2 units

Empty: 4 units silence

In advanced mode you can also choose the tempo of the melody (top slider), which is the speed at which it will be played.

Play SD Sound

Actuator: loud speaker

How to use it: use this action to play a sound that is available on a micro-SD card inserted in Thymio. Sounds on the micro-SD card need to be named “Px.wav” where x is the number entered in this icon. For instance this icon will play “P1.wav”.

Sounds are recorded at 8000Hz, mon, non compressed, Unsigned 8 bit PCM, wav format. The micro-SD card needs to be formatted in Fat32.

Stop SD Sound

Actuator: loud speaker

How to use it: use this action to stop a music playing.

Set LEDs

Actuator: 8 circular LEDs around Thymio buttons.

How to use it: use this action to set if the LEDs are on or off.


Yellow:  LED is on.

Dark grey: LED is off.

This action is often used to create state machines as you can then read the position of the LED (on/off) with a STATE block.

Selective set LEDs

Actuator: 8 circular LEDs around Thymio buttons.

How to use it: use this action to set if the LEDs are on or off.


Yellow:  LED is on.

Dark grey: LED is off.

Light grey : LED is ignored

This action is often used to create state machines as you can then read the position of the LED (on/off) with a STATE block. In advanced mode the “ignored” LED position allows you to create complex state machines.

Timer

This action does not use a specific actuator of the Thymio. It sets a timer inside Thymio. 

How to use it: this action sets a timer. Use the left digit to set the seconds and the right one to set the tenth of a second.

Compute

This action does not use a specific actuator of the Thymio. It sets a variable inside Thymio.

How to use it: this action is doing operations on the value of a variable inside Thymio. On the left you choose among a list of operators:

= Equals to
+ Plus

- Minus

x Multiply

÷Divide

On the right you choose a number from 0 to 255. This number is displayed on the circular LEDs as a binary number using the numbers like the image on the left.


The switch on the top left is for enabling and disabling the saturation of the binary number (when it reaches 255).
