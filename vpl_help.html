About Thymio

Thymio is a mobile robot designed for education. It has various sensors and actuators that can be programmed through different software.

For more information, visit www.thymio.org

About VPL

VPL is a block-based visual programming language. You can program behaviours for Thymio by associating events to actions.

Connecting a Thymio

Aseba 1.6.1 (and previous)

Plug your Thymio to your computer (with a USB cable or with a dongle), then turn it on.
Launch the Thymio Web Bridge inside your Aseba folder.

Thymio should now be detected by VPL 3, you are ready to go!

Thymio Suite

Plug your Thymio to your computer (with a USB cable or with a dongle), then turn it on.
Launch the Thymio Suite, Thymio should now be detected by VPL 3, you are ready to go!

Executing a program

To create a program simply drag one or more events to the left side of a line and then associate one or more actions you want to happen in the right side of the line. A line must always contain at least one event and one action.

If the code has no errors you can then execute it with the PLAY button.

The green PLAY button means the code has no errors, a robot is connected and the code in the interface is not yet inside the robot.

When you hit PLAY the code is sent to the robot, the button changes and you can REPLAY the same code whenever you want.

As soon as the code has been modified (or you clicked on STOP) the button goes back to the PLAY state.

Conditions and action blocks

Two types of conditions

On the left you have the events but more globally you have the conditions for the action to happen. You can have multiple conditions in one line of code but only the first bloc is considered an event. Other conditions are like states the program checks in order to trigger the action or not.  For instance, the following line means “when I hit the center button, if the color of the robot is green, then I switch the color to red”.

Note that depending on the order you place your different conditions the program may work or not as expected.

Some blocks are exclusively state blocks.

Events and states

Buttons

Sensor(s) used: five capacitive touch buttons on top of Thymio.

How to use it: this event detects if a button is pressed. If you want to detect if multiple buttons are pressed, you can combine multiple times the block inside the conditions.

Teal:  button is pressed.

Grey: button is not pressed.

Start of the program

This event does not use a specific sensor of the Thymio.

How to use it: this event is triggered at the start of the program, as you hit the PLAY button within VPL. You can use it to set an action Thymio needs to perform at the start of the program.
Example: if you want Thymio to move forward when you launch the program, associate this event to an action MOTORS block.

Orientation (basic)

Sensor(s) used: 3 axis accelerometer.

How to use it: This event detects Thymio’s orientation. In basic mode you can detect if Thymio is standing on the right side or if it is completely upside down.


Accelerometer (advanced)

Sensor(s) used: 3 axis accelerometer.

How to use it: This event detects Thymio’s orientation. In advanced mode you can detect any position on the three different axis.

Move the circular teal handle around the circle to choose the desired rotation. The orange arrow indicates the direction of gravity.

Tap (basic)

Sensor(s) used: 3 axis accelerometer.

How to use it: this event allows you to know when Thymio senses a shock. A tap with your hands is detected as a shock.

Ground detection (basic)

Sensor(s) used: ground proximity sensors.

How to use it: This event detects if there is a surface under Thymio.

Solid stroke means presence.

Dashed stroke means absence.

Note that a transparent or a black (matte) surface will be considered as absent.

Ground sensors (basic)

Sensor(s) used: ground proximity sensors.

How to use it:  This event allows you to separately read what the left and right sensors are detecting. They detect a black or white surface under Thymio. You can easily create a line following behaviour.

White: the sensor detects a white surface

Black: the sensor detects a black surface

Grey: the sensor is being ignored.

Note that black can also mean the absence of surface.

Ground sensors (advanced)

Sensor(s) used: ground proximity sensors.

How to use it: Ground sensors are proximity infrared sensors that detect the reflection of the surface they hit.

White: the sensor detects a reflection

Black: the sensor does not detect any reflection

Grey: the sensor is being ignored.

In advanced mode you can use a threshold (slider) to know which range of blacks or of whites is considered as black or white. The range of blacks or whites is illustrated on the surface facing the sensor. The threshold goes from 0 to 1000.

Note that a very reflective surface will be considered white and the absence of surface is considered as black.


Proximity sensors (basic)

Sensor(s) used: five proximity sensors at the front and two on the back.

How to use it: proximity infrared sensors measure the reflection of the surface in front of them. If there is a lot of reflection, this means that there is a close or very reflective obstacle in front of them. No reflection means either black or no obstacle at all.

White: an obstacle is detected (reflecting on the sensor)

Dash line:  no obstacle is detected (nothing is reflecting)

Grey: the sensor is being ignored.

Note that a transparent or a black (matte) object will be considered as absent.

Proximity sensors (advanced)

Sensor(s) used:five proximity sensors at the front and two on the back.

How to use it: proximity infrared sensors measure the reflection of the surface they hit. This way we know if there are obstacles around Thymio.

White: an obstacle is detected (reflecting on the sensor)

Dash line:  no obstacle is detected (nothing is reflecting)

Grey: the sensor is being ignored.

In advanced mode you can use a threshold (slider) to specify if the obstacle detected is rather close or far. The higher the threshold the closer to the sensor the detection is being done. The threshold goes from 1400 to 3300.

Note that a transparent or a black (matte) object will be considered as absent.
Different gray shades will be detected or not depending on the threshold.

Use cases:


Threshold: 3300

Obstacle: value > 3300
No obstacle: value < 3300

Purple: an obstacle is detected close to Thymio

Or
dashed line: no obstacle is detected close to Thymio        



Threshold: 1400
Obstacle: value > 1400
No obstacle: value < 1400


Purple: obstacle is detected far from Thymio
Or
dashed line: no obstacle is detected within a large distance


Timer up (advanced)

This event does not use a specific sensor of the Thymio. It reads a timer inside Thymio.

How to use it: this event detects when the timer (set in the actions) is over.

Variable (advanced)

This event does not use a specific sensor of the Thymio. It reads an internal state, a 8-bit variable inside Thymio.

How to use it: this event is reading the value of an internal variable. On the left you choose among a list of comparators:

= Equal to
≥ Bigger than

> Strictly bigger than

≤ Smaller than

< Strictly smaller than

On the right you choose a number from 0 to 255. This number is displayed on the circular LEDs as a binary number like the image on the left.



State Leds (basic)

This event does not use a specific sensor of the Thymio.  It reads the status of the 8 LEDs around Thymio’s buttons.

How to use it: this block allows you to know if these LEDs are on or off.

Teal:  LED is on.

Dark grey: LED is off.

You can use this to create basic state machines by associating the position (on or off) of the LEDs to a specific behaviour. By setting the LEDs with an action block you will then be able to switch between the different states. Since each LED can be either on or off there are 256 possible states.

State Leds (advanced)

This event does not use a specific sensor of the Thymio. It reads the status of some of the 8 LEDs around Thymio’s buttons.

How to use it: in advanced mode you can not only read if an LED is on or off but you can also choose to read only some LEDs, ignoring others.

Teal:  LED is on.

Dark grey: LED is off.

Grey stroke : LED is ignored
(Read LEDs are indicated with a purple point.)

This allows the creation of more complex state machines.

State: colors

This event does not use a specific sensor of the Thymio.  It reads status of the RGB LEDs displayed on Thymio, two on top and two in the bottom.

How to use it: use this block to check if Thymio RGB LEDs are set to a specific color. You can separately read the top and the bottom RGB LEDs.

This block allows also the creation of state machines as you can read the color (set by an action block) and then associate a behaviour to it. Every time you set a new color you are changing the state.

Actions

Directions (basic)

Actuator: setting the speed of the wheels

How to use it:  this action allows you to set a direction of movement for Thymio. Choose any arrow to make Thymio move or choose the center button to make it stop.

Note: with this block Thymio will always have the same speed (around 6-7 cm/s).

Wheels (advanced)

Actuator: setting the speed of the wheels

How to use it: this action allows you to set the speed of each wheel to make Thymio move. Each slider represents a wheel. The position on the slider indicates the speed of the wheel which goes from 0 to 500 (forward) and from 0 to -500 (backward). A speed of 500 is around 17cm/s.

To turn to a side simply set the speed of the opposite side wheel to a higher speed than the other. Depending on speeds Thymio will make narrower or wider curves.

Red square: Thymio is stopped.

Dark grey point: Speed is equivalent on both wheels.

Colors (basic)

Actuator: top and bottom RGB LED

How to use it: use this action to set a specific color for the Thymio RGB LEDs.

You can separately set the top and the bottom RGB LEDs.

Colors (advanced)

Actuator: top and bottom RGB LED

How to use it: use this action to set a color for the Thymio RGB LEDs. You can separately set the top and the bottom RGB LEDs.

In advanced mode you can precisely set the shade you want by combining the RED, GREEN and BLUE led values. This is the additive color mode, it means that when equal parts of red, blue, and green are added to one another, white light is created:

Music (basic)

Actuator: loud speaker

How to use it: use this action to make a melody with Thymio. Each blue line corresponds to a tone and you can choose the duration of the note by clicking on it:


Black: 1 unit

White: 2 units

Empty: 4 units silence

Music (advanced)

Actuator: loud speaker

How to use it: use this action to make a melody with Thymio. Each blue line corresponds to a tone and you can choose the duration of the note by clicking on it:


Black: 1 unit

White: 2 units

Empty: 4 units silence

In advanced mode you can also choose the tempo of the melody (top slider), which is the speed at which it will be played.

State Leds (basic)

Actuator: 8 circular LEDs around Thymio buttons.

How to use it: use this action to set if the LEDs are on or off.


Teal:  LED is on.

Dark grey: LED is off.

This action is often used to create state machines as you can then read the position of the LED (on/off) with a STATE block.



State Leds (advanced)

Actuator: 8 circular LEDs around Thymio buttons.

How to use it: use this action to set if the LEDs are on or off.


Teal:  LED is on.

Dark grey: LED is off.

Grey stroke : LED is ignored
(Read LEDs are indicated with a purple point.)

This action is often used to create state machines as you can then read the position of the LED (on/off) with a STATE block. In advanced mode the “ignored” LED position allows you to create complex state machines.

Timer (advanced)

This action does not use a specific actuator of the Thymio. It sets a timer inside Thymio. 

How to use it: this action sets a timer. Use the slider to choose the value you want, from 0 to 4 seconds.


Variable (advanced)

This action does not use a specific actuator of the Thymio. It sets a variable inside Thymio.

How to use it: this action is doing operations on the value of a variable inside Thymio. On the left you choose among a list of operators:

= Equals to
+ Plus

- Minus

x Multiply

÷Divide

On the right you choose a number from 0 to 255. This number is displayed on the circular LEDs as a binary number using the numbers like the image on the left.


The switch on the top left is for enabling and disabling the saturation of the binary number (when it reaches 255).
